# 金融 Agent 三工作台设计

## 目标

将当前 DeepSeek Agent Demo 扩展为面向 A 股、港股、美股和加密货币的金融研究 Agent。产品同时服务普通投资者与交易员/投研人员，提供行情研究、事件投研和交易员 Copilot 三个并列工作台，以及贯穿三个工作台的金融对话入口。

系统不提供自动下单或个性化投资建议。所有结论必须展示数据时间、来源、工具调用记录和风险提示。

## 用户与输出层级

### 普通投资者

- 以自然语言提问，例如“BTC 和 NVDA 今天为什么波动”。
- 首先获得简短结论、主要驱动、风险等级和需要关注的后续事件。
- 可展开查看报价、图表、新闻和来源，不需要理解工具或指标公式。

### 交易员与投研人员

- 可直接选择资产、周期、数据源和专业指标。
- 查看工具调用参数、原始数据时间、指标公式、置信度、事件关联资产与失效条件。
- 使用预警、研究记录和自选资产在三个工作台之间保持上下文。

## 信息架构

应用提供全局资产搜索、全局金融对话和三个主要工作台：

1. **行情研究**：报价、K 线、财务/估值指标、资产对比和由数据支持的问答结论。
2. **事件投研**：市场新闻、公司公告、财报和宏观事件的影响判断、关联资产和置信度。
3. **交易员 Copilot**：盘口或流动性、资金费率、未平仓量、链上/衍生品数据、预警和条件化交易假设。

自选资产、预警中心和研究记录是全局能力，不归属于单一工作台。对话面板在桌面端固定于工作台右侧，在窄屏端作为独立可展开区域；用户发起对话时，已选资产、当前工作台与时间范围自动作为上下文，但用户可在发送前删除上下文标签。

## 真实数据与金融对话

用户可以直接询问个股、币种、市场事件和授权研报，例如“腾讯和 BTC 今天上涨的原因是什么”“比较宁德时代与特斯拉的估值”或“这份研报的核心风险是什么”。

对话编排流程固定如下：

1. 识别市场、资产与问题类型；将资产名称映射为统一标识，例如 `600519.SH`、`0700.HK`、`AAPL.US`、`BTC/USDT`。
2. 根据问题选择工具：价格和图表使用行情工具，解释波动时补充新闻/公告，财务问题补充财报与估值，币种交易问题补充衍生品数据。
3. 工具网关取得最新允许使用的数据后，向 LLM 传入规范化数据快照、来源、数据时间、延迟、缺失项和工具调用摘要。
4. LLM 仅基于该快照形成结论；前端在回答下方展示实际调用的工具、来源链接、数据时间和风险提示。

输入区支持 `Enter` 发送、`Shift+Enter` 换行，并避免中文输入法候选确认触发发送。

## Agent 与工具架构

```text
用户问题或工作台操作
  -> Agent 编排器（意图识别、工具选择、结果聚合、回答分层）
  -> 工具网关（鉴权、限流、参数校验、缓存、审计）
  -> 数据适配器（股票、加密交易所、新闻/公告、链上或衍生品）
  -> 规范化市场数据与来源元数据
  -> Agent 回答 + 工具调用记录 + 风险提示
```

Agent 只能经由工具网关获取外部数据，禁止在没有工具结果时生成具体实时价格、涨跌幅、新闻事实或指标值。工具结果必须携带 `source`、`asOf`、`delay`、`symbol` 和 `confidence` 元数据。

一期工具接口定义如下：

| 工具 | 用途 | 主要输入 | 输出要求 |
|---|---|---|---|
| `get_quote` | 获取最新报价 | `symbol` | 价格、涨跌、市场、数据时间、延迟 |
| `get_candles` | 获取 OHLCV K 线 | `symbol`、`interval`、`limit` | K 线数组、时区、数据时间 |
| `get_company_financials` | 获取股票财务/估值数据 | `symbol`、`period` | 指标、报告期、披露日期、来源 |
| `search_market_news` | 搜索事件与新闻 | `query`、`from`、`to` | 标题、发布时间、发布方、链接、相关资产 |
| `get_crypto_derivatives` | 获取加密衍生品数据 | `symbol` | 资金费率、未平仓量、交易所、数据时间 |
| `calculate_indicator` | 基于已获取 K 线计算指标 | `candles`、`indicator`、`params` | 数值、公式、参数、使用数据范围 |
| `compare_assets` | 比较多个资产 | `symbols`、`metric`、`period` | 统一时间窗的对比结果和缺失说明 |

## 三工作台交互

### 行情研究

- 输入问题或选择一组股票/币种。
- Agent 先调用行情和 K 线工具；涉及解释时再调用新闻、财务或比较工具。
- 结果区上方呈现报价卡与简图，下方显示结论、风险和可展开的来源/调用记录。

### 事件投研

- 用户按资产、行业、时间窗或事件类型筛选。
- Agent 将新闻/公告与市场变动关联，并输出潜在影响、置信度和反证条件。
- 新闻只作为事实来源；“利多/利空”必须带有依据和不确定性，不得伪装为确定预测。

### 交易员 Copilot

- 用户选择加密资产、合约与时间周期；股票高级数据取决于授权的数据源。
- 输出流动性、资金费率、未平仓量、预警条件和策略失效点。
- 只提供研究与风险信息，不提供下单按钮、交易指令或收益承诺。

## 数据源与分层策略

- 一期开发与内测采用免 Key 的公开接口基线：Yahoo Finance Chart API 统一获取 A 股、港股和美股的报价与 K 线；东方财富公开行情接口补充 A 股；Binance Public REST 与 WebSocket 获取加密实时行情、K 线与公开衍生品数据。所有数据提供方经独立适配器接入，禁止在页面或 Agent 提示词中写死供应商逻辑。
- A 股、港股、美股的行情、财务数据和公告均使用具有明确商用授权和延迟说明的供应商；三个市场通过独立适配器接入，不混用代码、币种、交易时间或复权口径。接入供应商必须在采购/许可评审后确定，避免把免费开发者套餐误用于生产展示或再分发。
- 加密货币基础行情与实时/衍生品数据一期直接接入 Binance Public REST/WebSocket；后续可替换为具有再分发授权的聚合或交易所供应商。
- 新闻与公告优先接入授权 API 或官方披露源，不通过违反站点条款的抓取方式获取。研报只接入获得展示、检索和摘要授权的内容；无授权时只展示标题、来源链接和受许可的元数据，不存储或向 LLM 传递全文。
- 缓存层按数据性质设置 TTL：实时加密报价 1-5 秒、普通报价按供应商许可、K 线按周期、财务与公告按披露版本。页面始终展示数据时间与延迟。

## 安全、合规与质量

- 工具网关限制可访问资产和请求频率，拒绝模型构造的任意 URL、SQL 或代码执行请求。
- 保存每次工具调用的输入摘要、来源、时间、耗时、结果版本和错误码，敏感凭据永不回传给模型或前端。
- 对空数据、限流、过期数据和来源冲突展示明确状态；回答应说明无法确认而不是补全猜测。
- 提供统一风险免责声明，尤其在交易员工作台强调仓位、波动与失效条件。

## 实施顺序

1. 建立统一资产标识、市场数据模型、工具网关和可追溯的真实公开数据适配器；自动化测试使用固定回放数据，但用户请求只通过 Yahoo Finance、东方财富或 Binance 的真实响应处理。
2. 实现行情研究工作台及 `get_quote`、`get_candles`、`calculate_indicator` 和 `compare_assets`。
3. 接入新闻/公告数据，完成事件投研工作台和来源/置信度展示。
4. 接入有授权的加密衍生品数据，完成交易员 Copilot、预警和专业展开视图。
5. 在每个阶段加入数据质量抽样、供应商延迟监控、限流和降级验证。

## 验收标准

1. 三个工作台可在同一应用中切换，且共享资产搜索、金融对话、自选资产、研究记录与来源展示规则。
2. 任何含具体市场数据的回答都显示数据时间、来源和工具调用记录。
3. 当工具无数据、返回过期数据或发生错误时，Agent 不生成虚构数值或事实。
4. 普通视图默认显示结论、原因与风险；专业视图可展开参数、指标、原始来源和调用详情。
5. 系统无下单功能，不生成个性化买卖指令或收益承诺。
6. 用户针对 A 股、港股、美股、加密货币和授权研报的对话，均通过工具网关取得数据快照后再传给 LLM；前端可复核实际数据来源与调用时间。
