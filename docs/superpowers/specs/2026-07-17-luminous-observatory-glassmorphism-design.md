# Aurora Command 高端未来感视觉重构设计

## 背景与目标

`deepseek-agent-demo` 是一个包含会话、流式回答、金融研究模式、资产检索、联网搜索、离线重试与 WebSocket 状态的 AI 工作台。本次只重塑前端 HTML/CSS 的视觉呈现与交互反馈，不改变业务行为、接口、数据结构或现有组件语义。

目标用户是追求前沿科技体验的高端用户。界面应表现为克制、通透、可信的私人研究工作台：信息阅读优先，视觉氛围服务于专注，而非制造炫目的仪表盘。

## 选定方向

采用用户选定的「Aurora Command」方向：保留“左侧会话导航 + 顶部状态 + 居中对话/研究主区 + 底部输入器”的原始任务流；通过半透明玻璃表面、浅蓝至淡紫背景光、青紫单一强调渐变与柔和悬浮反馈建立高端未来感。视觉焦点是宽阔的研究阅读面、紧凑的数据溯源横条与一体化的底部输入停靠区。

## 视觉令牌

| 类别 | 规则 |
| --- | --- |
| 主背景（60%） | 柔白、冰蓝与淡紫的低对比渐变，叠加极弱噪点；四角仅使用大半径模糊光团。 |
| 辅助色（30%） | 雾蓝、灰紫、半透明白表面与深蓝灰文字。 |
| 强调色（10%） | 唯一高饱和青色到紫色线性渐变，用于主要操作、已选状态与关键 AI 标识。 |
| 字体 | `Inter` 优先，正文回退 `-apple-system` / 系统无衬线；全页不超过两套字体。 |
| 文字 | 标题 700、负字距；正文 400、行高约 1.8；数据/辅助信息保持清晰紧凑。 |
| 圆角 | 应用外壳和大区块 24px；按钮、筛选项和会话操作使用 100px 胶囊；内部数据元素 8px。 |
| 表面 | `backdrop-filter: blur(20px)`、半透明白底、必要时使用 `rgba(255,255,255,.3)` 高光边；其余分组用阴影与色差，而非灰色描边。 |

## 页面与组件规则

### 应用外壳与背景

- `body` 使用浅蓝—淡紫渐变，并以伪元素提供极轻的噪点纹理和缓慢漂移的三枚背景光团。
- `.app-shell` 为主玻璃容器，保留桌面双栏与现有移动端单列逻辑；不增加新页面或路由。
- 容器阴影采用大半径、低不透明度的蓝紫扩散阴影，确保白底文字仍保持 AA 对比度。

### 侧边栏、状态栏与导航

- 侧栏降低不透明实体感；会话行仍使用现有按钮和删除行为。
- 当前会话改为淡青紫玻璃底及渐变左侧光条；新建会话和金融对话保留原有文案与可访问名称。
- 顶部连接状态保留文字与圆点双重表达，不能仅靠颜色传达状态。

### 对话与金融工作台

- 消息使用轻玻璃分组与角色标记区分；保留 `assistant`、`user`、流式、停止、失败与工具溯源结构。
- 工具调用、市场研究状态和金融工作台模块采用低对比半透明表面，避免厚实边框与卡片嵌套。
- 主阅读列维持合理宽度；金融研究画布中的资产选择、标签和聊天入口继续沿用现有 DOM 与交互。

### 输入器与状态

- 输入器使用略高于消息层的玻璃表面与明确焦点光圈；现有 Enter 提交、Shift+Enter 换行、联网搜索、发送/停止状态均不变。
- 主按钮使用青紫渐变；危险和次级操作不引入第二套高饱和主题色。

## 交互与可访问性

- 可点击按钮、会话行及可交互卡片悬停时上浮 `translateY(-4px)` 并加深扩散阴影；按下时 `scale(.98)`，所有变化使用短促缓动。
- 保留 `:focus-visible` 焦点环、禁用态和移动端触控尺寸；减少动态效果偏好下禁用背景漂移、噪点动画和上浮过渡。
- 文本、状态和错误提示满足可读性要求；错误状态始终保留明确文字。

## 实施边界

- 主要修改 `src/styles.css`；仅在语义或现有选择器不足时对 JSX 增加最小类名，不调整业务逻辑。
- 不添加第三方 UI、图标或字体包，不修改服务端、IndexedDB、消息协议或测试逻辑。
- 验证覆盖现有 `pnpm test`、`pnpm build`、桌面与窄屏渲染，以及核心任务：登录、会话、发送/停止、金融模式和联网搜索开关。

## 自检

- 无 `TBD`、`TODO` 或悬而未决的视觉决策。
- 视觉方向与用户指定的浅色玻璃拟态、色彩比例、圆角层次、字体限制和交互状态一致。
- 实施范围明确限定为保留功能前提下的界面重构。
