# README 升级设计

## 目标

将项目根目录的 README 从简短的本地运行说明，升级为面向使用者和二次开发者的完整项目入口文档。读者应能在不通读源代码的情况下理解产品能力、运行边界、请求链路和主要模块职责，并可在本地安全启动与验证项目。

## 读者与范围

- **主要读者**：希望体验 Demo、接入 DeepSeek 配置或继续开发该项目的开发者。
- **次要读者**：需要快速了解功能、金融数据边界和安全策略的产品或评审人员。
- **包含内容**：项目概览、功能地图、交互工作流、架构和数据流、HTTP/SSE/WebSocket 接口、配置与运行、工具与数据源、安全边界、目录结构、测试和已知限制。
- **不包含内容**：修改业务代码、增加接口、承诺未实现的金融功能，或将演示性质的数据源表述为生产级服务。

## 文档结构

1. **项目概览**：说明这是 React、Vite、Express 与 DeepSeek 组成的金融 AI Agent 学习 Demo，并标出核心技术栈。
2. **功能地图**：按对话、金融工作台、本地持久化、实时连接、资产搜索和模型工具六个维度介绍能力；将尚未配置数据工具的工作台页签明确标为研究入口或占位状态。
3. **架构与数据流**：用 ASCII 图说明浏览器、Node 服务、DeepSeek、工具注册表、外部数据源和 IndexedDB 的关系；另以步骤说明聊天流、资产搜索、WebSocket 状态通知三条链路。
4. **接口契约**：列出健康检查、资产搜索、流式聊天和 WebSocket 的路径、方法、用途及关键事件；只记录当前实现的契约，不将内部实现细节伪装成稳定公开 API。
5. **运行与配置**：保留现有 pnpm 安装、`.env` 模板、启动命令和端口说明；配置表与 `.env.example` 保持一致，强调密钥不得提交。
6. **工具、行情与安全**：记录四个允许的模型工具、市场代码范围和数据源的演示性质；说明参数校验、调用上限、输出清洗、可信代理与 IP 定位条件。
7. **工程导览与验证**：按目录说明前端、服务端、市场模块、Agent 规划模块和测试；提供 `test`、`typecheck`、`build` 命令与预期用途。
8. **限制与免责声明**：明确仅供学习和研究，不构成投资建议，不执行交易；数据会延迟、缺失或不可用，生产使用须替换或评估外部服务。

## 架构表述准则

README 将描述真实的运行链路：前端通过 SSE 使用 `/api/chat/stream`，通过 HTTP 使用 `/api/market/search`，并与 `/ws` 维持状态连接；Express 服务把 DeepSeek 工具调用交给统一注册表，再访问天气、新闻、资产搜索和行情网关。

`server/agent/` 中的 LangGraph 规划图目前作为独立、受测试覆盖的规划模块存在，未接入 HTTP 聊天请求主链路。因此 README 将把它标为“可复用的规划模块”，不会称为“在线请求中的自动规划器”。

## 错误处理与安全表述

- 说明前端会显示流式请求失败，并将离线消息放入 IndexedDB 队列等待网络恢复后重试。
- 说明资产搜索支持请求取消；市场网关有请求超时、短期缓存、请求合并和供应商错误归一化。
- 仅列出可执行的四个具名工具；未知工具、非法参数、超出调用轮次或次数的调用均不能继续执行。
- 说明工具结果先经服务端字段清洗，再发送给模型与浏览器；新闻 URL 和内部异常不会透传。

## 验收标准

- README 中的所有脚本、环境变量、端口、路径、工具名和接口均可由当前仓库代码验证。
- 功能描述区分“已实现能力”与“研究入口/未配置能力”，不夸大实时性或交易能力。
- 新读者可据 README 在本地完成安装、配置、启动和基础验证。
- 文档不含密钥、个人信息或真实投资建议。

## 审校结论

已检查范围、运行链路和边界：本文档没有占位符或相互矛盾的描述；LangGraph 模块未被误写为在线聊天请求链路的一部分；金融工作台的未配置页签会在 README 中明确其当前状态。
