# DeepSeek Agent 页面视觉重构设计

## 目标

在不改变现有会话、登录、流式聊天、离线队列和 WebSocket 状态功能的前提下，重构 React 前端的视觉样式。新界面应当是紧凑、专业的工作台，而不是大字号的营销页或堆叠卡片界面。

## 范围

- 修改 `src/styles.css`，为登录页、应用外壳、侧边栏、状态栏、消息流和输入区建立统一视觉语言。
- 保留现有 `App.jsx` 与组件的行为、数据流和 CSS 类名；只在实现交互动效所必需时增加语义化类名或属性。
- 为鼠标与键盘用户提供明确的悬停、按下与焦点状态。
- 在系统开启“减少动态效果”时禁用非必要动画。

不包含：新增第三方 UI 库、修改服务端、重设 IndexedDB 数据结构、改变流式协议或增加产品功能。

## 视觉系统

- 字体优先使用 `"IBM Plex Sans"`，无本地字体时回退到系统无衬线字体；正文 13px，辅助文字 11-12px，页面标题不超过 20px。
- 页面底色使用暖灰白，内容区为白色，主文字为近黑色；仅使用深灰绿与低饱和绿色作为交互和在线状态强调色。
- 圆角分层：应用外壳 18px，输入区 15px，按钮和会话行 10-11px，头像 8px；避免大面积胶囊形卡片。
- 采用 8px 间距网格，并保留侧栏 280px、主内容自适应的桌面布局；窄屏下侧栏转为顶部紧凑区或按现有单列规则堆叠。

## 布局与组件

- 登录页：保留单一登录面板，缩小标题和面板留白，使用轻微的暖灰到浅绿背景光晕。
- 应用壳：将主界面放入带 18px 圆角、细边框和低阴影的容器，留出桌面边距；移动端取消外侧边距以避免浪费空间。
- 侧边栏：使用浅色中性背景。新建会话使用深色实心按钮；当前会话用淡绿底色和短左侧指示条标识，列表项悬停时向右轻移 2px。
- 状态栏：使用低调的在线圆点与短文本；错误消息维持可读的红色强调，不使用抢眼的整条背景。
- 消息区：主内容限制阅读宽度，消息不再使用厚边框卡片；用户和 AI 通过头像、角色标签、轻量底色和代码块区分。
- 输入区：固定在消息区底部，使用 15px 圆角和清晰焦点环；发送和停止操作维持现有功能与可访问名称。

## 动效：流式聚焦

- AI 输出开始时，显示低强度的绿色径向光晕；新到达的文本块以 180ms 的淡入上移呈现，不为历史消息反复播放。
- 生成中的状态以短闪烁光标或三点提示显示；流式结束、失败或被停止时立即清除。
- 会话切换和列表悬停使用 160-220ms 的 `ease-out` 颜色与位置过渡；发送按钮可上移 2px，但没有持续循环。
- 所有循环动画仅用于在线状态和生成中提示，持续时间不少于 1.6 秒；`prefers-reduced-motion: reduce` 下禁用转换、光晕和循环动画。

## 响应式与可访问性

- 桌面主内容最大宽度保持在 760px 左右，保证长回复可读。
- 800px 以下保持现有单列逻辑，侧栏不挤压输入区；最小触控目标为 40px，高频按钮在移动端不小于 44px。
- 所有按钮和输入控件保留至少 3px 的可见焦点环；在线颜色不作为唯一状态信息。
- 维持文字与底色至少 4.5:1 的对比度，代码块可横向滚动。

## 验收标准

1. 现有 `pnpm test` 全部通过，且 `pnpm build` 成功。
2. 登录、创建会话、删除会话、清空会话、发送、停止、离线提示和 WebSocket 状态仍可用。
3. 常规正文为 13px，辅助文字不大于 12px，页面不存在超过 20px 的大标题。
4. 桌面与 800px 以下视图都不会产生横向溢出；输入区和操作按钮仍可触达。
5. 流式生成、会话悬停和发送按钮有上述限定的动效；减少动态效果偏好下不播放装饰动效。
